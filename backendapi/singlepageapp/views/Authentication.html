OAuth is the main authorization framework used for AHMC API. To know and read more about OAuth, refer to RFC 6749 .


Below is the Abstract Protocol Flow of OAuth.




Token Based Authentication
An access token will be used for authenticating a user for every request sent to the server.




Get Access and Refresh Token

STEP 1 Send a POST request to the Authorization Server together with the given credentials, client_id and client_secret.


STEP 2 Once request is successful, an access and refresh token will be sent by the Authorization Server.




STEP 3 The access token must be included in the Authorization header for every request to the resource server.


STEP 4 Request another access token to the authorization server using the refresh token.




Take Note:
1. Access Token expires after 1 hour while refresh token lasts for 24 hours.
2. client_id and client_secret needs to be included for every request sent to the authorization server.
